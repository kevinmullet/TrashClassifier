# ♻️ Trash Classifier — SVM (Paper / Glass / Plastic)

A classic computer-vision image classifier that sorts trash photos into
**paper**, **glass**, or **plastic**. No deep learning — it turns each image
into hand-crafted features (color, texture, shape, reflectivity) and feeds
them to an RBF-kernel Support Vector Machine.

Built on a balanced 3-class subset of
[TrashNet](https://github.com/garythung/trashnet).

## Results

Measured on a held-out, stratified **15% test set** (237 images) that the model
never sees during training or tuning.

| Metric | Score |
|---|---|
| Accuracy | **0.894** |
| F1-score (macro) | **0.891** |
| F1 — paper | 0.939 |
| F1 — glass | 0.880 |
| F1 — plastic | 0.853 |
| Inference latency | **~57 ms / image** (CPU) |

`models/classification_report.txt`, `confusion_matrix.png`, and
`learning_curve.png` are regenerated on every training run.

## How it works

Each image is resized to 224×224 and converted into one feature vector built
from eight blocks:

- Color histogram (BGR + HSV)
- Color moments (mean / std / skewness)
- LBP and GLCM (texture)
- HOG (shape, 32×32 cells)
- Edge density (Canny + Sobel)
- Specular highlights (reflective glass vs. opaque plastic)
- Spatial-pyramid region color (color *and where it sits* in the image)

The classifier is an **RBF SVM** (one-vs-one). Because HOG is far larger than
the other blocks, a `ColumnTransformer` compresses HOG on its own (PCA) while
the remaining blocks keep full weight, so color and reflectivity are not
drowned out. Hyperparameters are tuned with **grouped 5-fold GridSearchCV**
(`StratifiedGroupKFold`) scored on macro-F1 — grouping keeps an image and its
augmented copies in the same fold so the cross-validation score is honest.

The training set is augmented (flip, rotate ±20°, zoom, brightness, contrast,
blur); validation and test sets are left untouched.

## Project structure

```
.
├── app.py                # Streamlit web app (loads the trained pipeline)
├── train_model.ipynb     # data -> features -> train -> evaluate -> save
├── requirements.txt
├── paper/ glass/ plastic/ # dataset folders (not committed — see below)
└── models/               # generated by training (not committed by default)
    ├── pipeline.joblib            # the full ColumnTransformer -> SVC model
    ├── label_encoder.joblib       # class index <-> name
    ├── test_split.joblib          # held-out test features + labels
    ├── classification_report.txt
    ├── confusion_matrix.png
    └── learning_curve.png
```

The dataset folders and the trained `models/` are **not tracked in git**
(see `.gitignore`) — the dataset is large and the model is reproducible from
the notebook.

## Setup

Requires Python 3.10+.

```bash
pip install -r requirements.txt
```

Download the TrashNet `paper/`, `glass/`, and `plastic/` image folders into the
project root (each containing `.jpg` files).

## Usage

### 1. Train the model

```bash
jupyter notebook train_model.ipynb
```

Run all cells. The notebook loads ~1,577 images, splits them 70/15/15
(stratified), extracts features (the slow step), runs the grouped GridSearchCV,
evaluates on the test set, measures latency, plots the learning curve, and
writes everything to `models/`.

### 2. Run the web app

```bash
streamlit run app.py
```

Open `http://localhost:8501`, upload a JPG/PNG of trash, and the app shows the
predicted class (Indonesian + English), confidence, per-class probability bars,
and the inference time. It refuses to start with a clear message if
`models/pipeline.joblib` is missing — train the notebook first.

## Deployment (Hugging Face Spaces)

The YAML header at the top of this file configures a Streamlit Space.

1. Create a new Space with SDK **Streamlit**, or link this GitHub repo.
2. Hugging Face auto-builds from `app.py` and `requirements.txt`.

Training does **not** run on the Space, so the app needs a pre-trained model
present. `models/` is gitignored by default; to deploy, commit the two files
the app actually loads:

```bash
git add -f models/pipeline.joblib models/label_encoder.joblib
git commit -m "Add trained model for deployment"
```
